# 커밋 컨벤션

## 형식

커밋 메시지는 아래 형식을 사용한다.

```text
<type>(<scope>): <subject>
```

`scope`가 명확하지 않으면 생략할 수 있다.

```text
<type>: <subject>
```

## Type

- `feat`: 새로운 기능
- `fix`: 버그 수정
- `docs`: 문서 변경
- `test`: 테스트 추가 또는 수정
- `refactor`: 동작 변경 없는 구조 개선
- `chore`: 빌드, 설정, 의존성 등 기타 작업
- `ci`: CI/CD 변경

## Subject

- 한국어 또는 영어를 사용할 수 있다.
- 최근 커밋 스타일을 우선 참고한다.
- 한 문장으로 변경 의도가 드러나야 한다.
- 마침표로 끝내지 않는다.

## Commit Unit

- 하나의 커밋은 하나의 논리적 변경만 담는다.
- 문서 변경과 테스트 코드 변경처럼 목적이 다르면 커밋을 분리한다.
- 설정 변경과 기능 변경처럼 되돌리는 이유가 다를 수 있으면 커밋을 분리한다.
- 자동 배포 이미지 태그 변경은 기존 스타일을 따라 `🤖 ci(deploy): update traffico-api image tag`를 사용한다.
