# 커밋 생성

## 절차

1. `git status`와 `git diff`로 변경사항 확인
2. `git log --oneline -5`로 최근 커밋 스타일 참고
3. `.Codex/rules/commit-convention.md` 컨벤션에 따라 커밋 메시지 작성
4. 변경사항을 스테이징하고 커밋
5. 변경사항이 여러 논리적 단위로 나뉘면 커밋을 분리한다

## 커밋 단위
- 하나의 커밋 = 하나의 논리적 변경 (한 문장으로 설명 가능해야 함)
- 새 서비스 추가 시: Application 매니페스트 + Helm 차트를 같은 커밋에 포함
- 설정 변경과 구조 변경은 분리
- 변경사항이 여러 논리적 단위로 나뉘면 커밋을 분리한다
